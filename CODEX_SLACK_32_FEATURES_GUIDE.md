# Codex 연동 기준 Slack 반영 가능 32개 기능 가이드

이 문서는 현재 권한 기준 32개 기능에 대해 다음을 정리합니다.
1. Codex에 자연어로 어떻게 요청하면 되는지
2. 실행 후 Slack에서 어떤 변화가 보이는지

중요:
- 아래 `메서드` 열은 내부 매핑용 참고 정보입니다.
- 실제 사용자는 메서드명을 쓰지 않고 자연어로 요청해도 됩니다.
- 권한 충돌을 피하려면 문장에 `USER 토큰으로`를 같이 적는 방식을 권장합니다.

---

## 공통 자연어 요청 템플릿

1. `USER 토큰으로 <채널ID> 최근 10개 메시지 읽어서 요약해줘.`
2. `USER 토큰으로 <채널ID>에 "<메시지>" 전송해줘.`
3. `캔버스 "<제목>" 하나 만들어줘.`
4. `실패하면 Slack error/needed/provided 값을 그대로 보여줘.`

## 1) Canvases (6)

| 메서드(내부 매핑) | 자연어 요청 예시 | Slack에서 보이는 변화/확인 포인트 |
|---|---|---|
| `canvases.create` | `"MCP 테스트 캔버스"라는 제목으로 새 캔버스 만들어줘.` | 새 캔버스가 생성됩니다. |
| `canvases.edit` | `캔버스 ID가 <ID>인 문서 본문을 회의 템플릿으로 바꿔줘.` | 기존 캔버스 내용이 수정됩니다. |
| `canvases.delete` | `캔버스 ID <ID> 문서 삭제해줘.` | 캔버스가 삭제되어 접근이 안 됩니다. |
| `canvases.access.set` | `캔버스 ID <ID>를 특정 사용자/그룹이 볼 수 있게 권한 추가해줘.` | 지정 대상의 접근 권한이 생깁니다. |
| `canvases.access.delete` | `캔버스 ID <ID>에서 특정 사용자/그룹 권한 제거해줘.` | 지정 대상의 접근 권한이 사라집니다. |
| `canvases.sections.lookup` | `캔버스 ID <ID>에서 "액션 아이템" 섹션 찾아줘.` | 화면 변화 없이 섹션 조회 결과가 반환됩니다. |

## 2) Chat (11)

| 메서드(내부 매핑) | 자연어 요청 예시 | Slack에서 보이는 변화/확인 포인트 |
|---|---|---|
| `chat.startStream` | `<채널ID> 채널에 실시간 업데이트용 스트리밍 메시지 시작해줘.` | 스트리밍 메시지 세션이 시작됩니다. |
| `chat.appendStream` | `방금 시작한 스트리밍 메시지에 진행 상황 한 줄 더 붙여줘.` | 같은 스트림 메시지 내용이 이어서 갱신됩니다. |
| `chat.stopStream` | `현재 스트리밍 메시지 마무리하고 종료해줘.` | 스트리밍 업데이트가 종료됩니다. |
| `chat.delete` | `<채널ID> 채널의 ts=<메시지ts> 메시지 삭제해줘.` | 해당 메시지가 채널에서 사라집니다. |
| `chat.deleteScheduledMessage` | `예약된 메시지 ID <scheduled_message_id> 취소해줘.` | 예약 큐에서 제거됩니다. |
| `chat.getPermalink` | `<채널ID> 채널 ts=<ts> 메시지 링크 알려줘.` | 메시지 퍼머링크 URL이 반환됩니다. |
| `chat.meMessage` | `USER 토큰으로 <채널ID>에 "/me 테스트 중" 형태 메시지 보내줘.` | 채널에 me 스타일 메시지가 표시됩니다. |
| `chat.postEphemeral` | `<채널ID> 채널에서 유저 <USER_ID>에게만 보이는 임시 메시지 보내줘.` | 대상 사용자 화면에만 임시 메시지가 보입니다. |
| `chat.scheduledMessages.list` | `내 예약 메시지 목록 보여줘.` | 화면 변화 없이 예약 목록이 반환됩니다. |
| `chat.scheduleMessage` | `USER 토큰으로 <채널ID>에 10분 뒤 "테스트 메시지" 예약 전송해줘.` | 지정 시각에 메시지가 자동 게시됩니다. |
| `chat.update` | `<채널ID> 채널 ts=<ts> 메시지를 "수정된 문구"로 바꿔줘.` | 기존 메시지 내용이 수정됩니다. |

## 실패 줄이기 체크리스트

1. 요청 문장에 토큰 힌트를 명확히 넣기: `USER 토큰으로 ...`
2. 대상 채널/유저/ts 값을 함께 넣기
3. 실패 시 `error`, `needed`, `provided` 원문 요청하기
4. `not_in_channel`이면 해당 채널에 앱/토큰 주체가 참여 중인지 먼저 확인
5. `missing_scope`이면 필요한 scope 추가 후 토큰 재발급 및 재설치

## 바로 복붙 가능한 자연어 예시

1. `USER 토큰으로 C0AHJ8GF09H 채널 최근 10개 메시지를 읽고 5줄로 요약해줘.`
2. `C0AHJ8GF09H 채널 목록 정보와 멤버 수를 표로 보여줘.`
3. `USER 토큰으로 C0AHJ8GF09H 채널 ts=1730000000.123456 메시지를 "테스트 수정본"으로 바꿔줘.`
4. `"MCP 점검용 회의록" 캔버스 하나 만들고 생성된 ID도 같이 알려줘.`
